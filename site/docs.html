CLI

The daily moves, source-checked.

Session IDs can be full IDs or unique prefixes from sessions ls. Put the global --json flag before the command when a script needs structured output.

sessions new --tool claude|codex|shell

Start an interactive session. Add --cwd, --name, --description, --model, --effort, or --on-idle as needed.

sessions ls [--mine]

List live sessions, scoped to agent provenance with --mine. Add -a to include exited sessions.

sessions list --mine

List agent sessions and headless lanes together. Purpose descriptions make cleanup decisions readable.

sessions send <id> "…"

Send text plus Enter, then confirm that Claude or Codex received it. Add --file path for a prompt file or --no-wait for fire-and-forget.

sessions ask <id> "…"

Send with confirmation, wait for the working → idle edge, and print the last assistant reply.

sessions wait <id>

Block until a session is idle or a lane exits. Tune with --idle and --timeout; condition waits can watch a commit, file content, or stable idle period.

sessions run [options] -- <command…>

Start a supervised, headless lane. Add --name, --description, --cwd, or --spec; add --wait --output to block and print its captured tail.

sessions lanes

List running and completed lanes with exit codes, durations, working directories, and names.

sessions search <query> [--ranked]

Search live and persisted chat history. Substring is the precise default; choose regex or FTS5-ranked stemming, phrases, and boolean operators.

sessions status <id>

Show a compact session card with activity, git state, runtime state, and the latest producer verdict.

sessions verdict <id>

Read the latest explicit verdict. Use sessions verdict emit <id> with schema-versioned JSON to append one.

sessions recover [--reopen]

Inspect unexpectedly lost lanes, then explicitly reopen the recoverable ones.

sessions adopt <path-or-uuid>

Bring an existing Claude or Codex provider session under sessions's supervision.

sessions move <id> --to <endpoint>

Prepare and resume a session on another sessions daemon. Preview with --dry-run; dirty worktrees require --allow-dirty.

sessions lan enable|status|disable

Opt into a second HTTP listener on the current same-Wi-Fi or Ethernet address. Protected routes still require authentication.

sessions remote enable|status|disable

Manage the tailnet-only Tailscale Serve HTTPS endpoint used for direct remote access.

sessions pair [--name NAME]

Mint a five-minute, single-use QR ticket for the best enabled endpoint. The device gets its own revocable token; the master token is not embedded.

sessions devices [revoke <id>]

List per-device credentials or invalidate one immediately by exact ID or unique prefix.

sessions backup enable --project P --encrypt

Encrypt transcripts and the manifest locally with XChaCha20-Poly1305 before direct upload to your own somewhere project. Use now, status, or decrypt afterward.

sessions models

List the live Codex app-server model catalog, including supported efforts and service tiers.

sessions model <id> <model>

Switch the model, and optionally effort, for an idle supported session.

sessions kill <id> […]

Terminate one or more sessions and remove their supervised runners.

sessions doctor

Check the daemon and per-session health; flag runners that need attention.

sessions support [--diagnostics]

Open the official feedback, bug-ticket, and private security-report channels. The optional local preview excludes session content, IDs, paths, credentials, environment, logs, and crash files; nothing is uploaded automatically.

sessions install

Register sessionsd as the macOS LaunchAgent, start it, and check its health.

sessions uninstall

Stop the daemon and remove its launchd registration without deleting session or recovery data.

agent controls

Choose the gear.

--model <name>

Choose the startup model for Claude or Codex.

--effort <level>

Pass the requested reasoning effort to the agent.

--fast

Use Codex priority service tier. Claude has no equivalent, so sessions rejects that combination.

$ sessions models
$ sessions new --tool codex --model <model> --effort high --fast --cwd ~/work

when work stops

Let the session tap you.

Put one global shell command in ~/.config/sessions/hooks.json. It runs on each observed working → idle edge, in that session's working directory.

{
  "onIdle": "~/bin/notify-sessions"
}

The hook receives these environment variables. SESSIONS_OUTCOME is done, blocked, or error; a missing final assistant summary is an empty string.

SESSIONS_SESSION_ID SESSIONS_SESSION_NAME SESSIONS_SESSION_TOOL SESSIONS_SESSION_CWD SESSIONS_OUTCOME SESSIONS_FINAL_MESSAGE SESSIONS_DURATION_MS
🟢 done

The agent reached a clean finish; the notification body carries its final summary.

🟡 needs you

A prompt, permission request, or visible choice is waiting for a person.

🔴 error

The final screen still shows an unresolved failure.

Classification and summary extraction are best-effort. Browser notifications and idle hooks exist today; native Android push when an agent needs you is still being built. Use sessions help as the final word on command syntax. Need the walkthrough? Get set up →

the spine

Local by default.
Direct by design.

No Sessions account is required, and Sessions never relays terminal bytes. Localhost is zero setup; LAN is an explicit opt-in; remote access stays inside Tailscale and terminates TLS on your Mac. Optional encrypted backup is the exception that leaves the machine, and it leaves as ciphertext sent directly to your own somewhere project. Sessions is free; somewhere is the platform behind it.
