get set up

Let's get you running.

Two ways in — on your Mac, or from anywhere. Both take minutes. And this page checks each step actually worked, so you're never guessing whether it's connected.

0
Before you start
a 10-second inventory

You need a Mac, Node 18+ (node -v), and Claude Code and/or Codex already installed — pretty runs the agents you already have; it doesn't replace them.

1
Install pretty
on your Mac
early access — CLI landing soon
npm i -g pretty-pty
pretty install

pretty install registers a small background daemon that survives restarts and reboots, then starts it. Your sessions live in it — close terminals, close the laptop lid, they keep going.

Open it on this Mac →
Stuck on install? If npm errors about node-pty / a C++ build, run xcode-select --install first, then retry. Any other trouble: pretty doctor tells you exactly what's wrong.
Installation troubleshooting
node-pty or C++ build failed

Install Apple's command-line tools with xcode-select --install, then run the npm install again.

The daemon is down

Run pretty doctor. It checks the native PTY, daemon, and session runners and tells you what needs attention.

2
Reach it from anywhere
optional · +2 minutes

Want your sessions on your phone, or from another laptop? You'll put them on Tailscale — a private network just for your devices. No ports opened to the internet, and nothing routed through us: your browser talks straight to your Mac.

a. Install Tailscale and sign in — tailscale.com/download. Do it on your Mac and your phone (same account).

b. Back on your Mac, turn on remote access:

pretty remote enable

That runs tailscale serve for you (gives your daemon a real HTTPS address like https://your-mac.tailnet.ts.net) and prints a QR code. Scan it with your phone — it lands you right back here, already connected, no typing.

One honest heads-up: Tailscale's HTTPS certificate lists your machine + tailnet name in public certificate-transparency logs. Harmless for most, but if your machine name is sensitive, rename it before you enable serve.
Remote-access troubleshooting
The .ts.net address does not resolve

Let this device use Tailscale DNS with tailscale set --accept-dns=true. Then confirm MagicDNS is enabled in the Tailscale DNS console and retry pretty remote status.

The address resolves but does not answer

Make sure both devices are signed into the same tailnet, then run pretty doctor and pretty remote status on your Mac.

not checked yet

On your Mac? Just try http://localhost:8787. From your phone? Use the .ts.net address the QR gave you.

leave no daemon behind

Uninstall cleanly.

End any sessions you still need first. Then stop the daemon, remove its login item, and uninstall the package. Each line is separate so it is safe to copy and run one step at a time.

launchctl bootout gui/"$(id -u)" "$HOME/Library/LaunchAgents/tech.pretty-pty.daemon.plist"
rm -f "$HOME/Library/LaunchAgents/tech.pretty-pty.daemon.plist"
npm rm -g pretty-pty
Close the laptop. Take the walk.
It keeps going.
