already shipped

The feature tour is real.

Not a waitlist and not a mockup. These are the parts already in the Go runtime and CLI today.

i.

Never lose the fleet

Each session lives in its own runner, outside the daemon. The ledger records what exists, so reloads do not kill the work and sessions recover --reopen can rebuild a lost lane in one command.

ii.

Structured, not scraped

Codex runs through its JSON-RPC app-server contract; Claude runs through its structured stream-JSON path. You get reliable sends, streaming, and model controls without driving a TUI by screen-scraping.

iii.

Search all of it

sessions search covers live and persisted conversations. Keep exact substring and regex for precision, or opt into FTS5 ranking with BM25, stemming, phrases, and boolean queries.

iv.

Access grows with you

Use localhost with zero setup, sessions lan enable on the same Wi-Fi without Tailscale, or sessions remote enable anywhere over Tailscale Serve. Remote TLS ends on your Mac.

v.

Scan once. Revoke anytime.

sessions pair mints a five-minute, single-use QR ticket. The device receives its own revocable credential; the daemon's master token is never put in the QR.

vi.

Encrypted before upload

sessions backup … --encrypt applies XChaCha20-Poly1305 on your Mac before your history reaches your own somewhere project. The recovery key stays with you; nobody without it — including us — can read the backup.

Built for agents running agents: install the bundled sessions skill, use --mine to query work by provenance, and add --description so every lane says why it exists. If you omit one, sessions can derive the purpose from the first message.

Feature sources: docs/WHY.md, docs/CLI.md, runtime/cmd/sessions/help.go, runtime/internal/api/pair.go, and runtime/internal/backup/encrypt.go.

the contract

What we never do

This page is static files. Your terminal traffic flows directly between your browser and your Mac. That architecture makes these promises structural, not aspirational.

see your data

There is no Sessions relay and no hosted session store. Terminal bytes travel browser → your Mac over loopback, your local network, or your own tailnet. We never see, store, or forward a single keystroke or token of output.

hold your keys

QR pairing puts a five-minute, single-use ticket in a URL fragment — the part browsers never send to a server. Your Mac exchanges it for one revocable device credential. The master daemon token is never in the QR.

kill your sessions

A bad deploy of this page cannot touch your running agents — separate runner processes own the work on your machine, and the daemon serves its own UI at localhost:8787 as the permanent local path.

phone home

No analytics, no telemetry, and no Sessions account required. Optional remote access uses your Tailscale network; optional backup writes directly from your daemon to your own somewhere project.

run install scripts

Each piece is a single static Go binary — CLI, daemon, and runner — and the release archive contains those same auditable files. No Node runtime, no npm, no native-module build, and no script piped from the internet.

ship what we don't run

We run our own agent work on the same runner, daemon, ledger, and CLI documented below. If the durable-session path breaks, it breaks our work first.

Precise version: the core runtime is local and terminal traffic has no Sessions data plane. Remote use is strongest through the daemon-served same-origin UI; the hosted onboarding page is still mutable web code. Encrypted backup protects bytes before upload. That honesty is part of the contract too.

# Homebrew — no Node, npm, or install script:
$ brew install uzihaq/tap/sessions
$ sessions install  # registers the daemon, prints your local URL
# or download the static archive directly:
$ curl -fLO https://github.com/uzihaq/sessions/releases/download/v<version>/sessions_<version>_darwin_arm64.tar.gz
$ sessions pair  # optional — one-time QR, per-device credential
