what it is

tmux, if it were built for agents

i.

Sessions are sacred

Every session is its own supervised process, owned by macOS launchd — not by a terminal window. Daemon restarts, app updates, reboots: your agents reattach and continue.

ii.

Sends that confirm

The worst agent bug is the message that never arrived. pretty confirms every send against the agent's own transcript — it lands, or it fails loudly and keeps your draft.

iii.

Two views, one truth

A raw terminal when you need the cockpit, a clean chat view when you're reading on your phone. Both derived from the same session — nothing re-typed, nothing scraped.

the contract

What we never do

This page is static files. Your terminal traffic flows directly between your browser and your Mac. That architecture makes these promises structural, not aspirational.

see your data

There is no relay and no backend session store. Terminal bytes travel browser → your Mac over your own tailnet. We never see, store, or forward a single keystroke or token of output.

hold your keys

Your daemon token is generated on your Mac and shared via a URL fragment — the part of a link that browsers never send to a server. It cannot appear in our logs, because it never reaches us.

kill your sessions

A bad deploy of this page cannot touch your running agents — they live in launchd on your machine, and the same UI is served locally at localhost:8787 as a permanent fallback.

phone home

No analytics, no third-party scripts, no telemetry. View source on this page — it's one HTML file. What you can read is all there is.

ship what we don't run

We run our own fleet — dozens of concurrent agent sessions, every day — on exactly the setup documented below. If it breaks, it breaks us first.

Precise version: the hosted page is the terminal client, so "never" means never by architecture — no relay, no storage, no third-party code — not a cryptographic impossibility. That honesty is part of the contract too.

  2. 2pretty install

From anywhere

Your phone, your laptop, the café — over your own private tailnet.

  1. 1Install Tailscale and sign in (free)
  2. 2pretty remote enable
  3. 3Scan the QR it prints — sessions on your phone.
Set up phone access →
# Homebrew — no Node, npm, or install script:
$ brew install uzihaq/tap/pretty
$ pretty install  # registers the daemon, prints your local URL
# or download the static archive directly:
$ curl -fLO https://github.com/uzihaq/pretty-PTY/releases/download/v<version>/pretty-pty_<version>_darwin_arm64.tar.gz
$ pretty remote enable  # optional — phone access via Tailscale
